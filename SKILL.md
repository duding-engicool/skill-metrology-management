# 计量管理

## ① 角色定位

- **计量管理员**：负责建立并维护全公司计量器具台账，安排周期性校准/检定，跟踪证书有效期。
- **实验室主管 / 质检负责人**：需要掌握在用量具合规状态，审核迎检前确认无超期器具。

使用者通常面临器具数量多、分散在各车间、校准周期不一、靠手工 Excel 推算易漏易错的问题。本技能替其完成台账结构化、周期自动推算、超期预警，实质校准结论仍以证书为准。

## ② 使用场景

| 触发条件 | 手头材料 | 典型诉求 |
|----------|----------|----------|
| 每月/每季盘点计量器具 | 现有台账或器具清单 | "本月有哪些要校准？" |
| 新增一台设备需入台账 | 器具名称、量程、精度、上次校准日 | "把这台千分尺加进台账" |
| 审核前合规自查 | 器具清单+证书 | "生成计量合规状态报告" |
| 发现某器具超期 | 器具编号 | "这台是不是超期了？" |

触发关键词：计量台账、校准周期、下次校准日期、超期预警、送检计划、计量合规。

## ③ 解决的问题

- **台账混乱**：器具分散、信息不全，无法一眼看清状态。
- **超期未校**：手工推算易漏，超期器具仍被用于检验，带来合规风险。
- **周期计算靠手工**：不同器具周期不同（3/6/12 月等），人工加月份易错。

本技能不替代计量检定机构出具证书，也不替代强制检定设备的法定程序（强检设备超期直接判定不合格）。

## ④ 能力边界

**能做**
- 维护计量器具台账（新增/更新/查询）。
- 依据"上次校准日期 + 校准周期"自动计算下次校准日期。
- 三级预警：红色超期 / 橙色 7 天内到期 / 黄色 30 天内到期 / 正常。
- 生成 **MD 管理台账**（默认）；生成后**主动询问用户是否需要 Word / Excel 版本**，确认后导出。
- 提示 MSA（测量系统分析）关联待办。

**不做**
- 不替代计量机构出具校准/检定证书。
- 不判定器具本身合格与否（以证书结论为准）。
- 不编造证书编号、机构名称；不确定处标注「待企业补充」。
- 不强制交互锁：信息不全时以合理默认值（周期按器具类别推荐）填充并标注【待填写】。
- 不主动生成网页版（无 HTML 输出）。

**输入要求**
- 至少提供器具名称；推荐提供上次校准日期与校准周期，缺失则按类别默认周期并标注。

## ⑤ 交互与角色设定

- **语气**：专业、务实、客观，避免空话；用语推荐"标识、隔离、停用、送检、追溯"。
- **采集方式**：先确认动作（查询/新增/合规检查/送检计划），再收集器具字段；信息不全不强制追问，按类别默认周期填充并标注。
- **是否强制交互锁**：否（生成/台账类）。允许直接出框架或合规报告。
- **引导深度**：中等。给出字段模板与默认值，明确标注需人工确认项，不替使用者下检定结论。
- **合规红线**：超期器具必须提示"立即停用"，强检器具超期直接提示"判定不合格，安排检定"。

## ⑥ 处理流程

### 步骤 1：识别动作
- 查询到期 → 按时间范围（本月/下月/本季）列预警清单。
- 单台查询 → 按编号或名称查档案与下次到期日。
- 合规检查 → 全量合规率 + 风险项 + MSA 待办。
- 送检计划 → 按月汇总送检清单（建议日期/机构/周期）。

### 步骤 2：维护台账
- 新增：名称、厂商、量程、精度、工位、上次校准日、周期（可选，按类别推荐）。
- 更新：记录本次校准完成，自动重算下次到期日，状态转"合规"。

### 步骤 3：周期计算与预警
- 下次校准日期 = 上次校准日期 + 校准周期（月）。
- 与当前日期比较，判定 红/橙/黄/正常 四级。

### 步骤 4：渲染报告并确认导出格式
- 默认生成 **MD 管理台账**：`python scripts/build_report.py`，输出到**用户当前工作目录**。
- 生成后**主动询问用户**："是否需要 Word 或 Excel 版本的台账？"
  - 仅 MD（默认即可，不再额外操作）
  - Excel → `python scripts/build_report.py --format xlsx`
  - Word  → `python scripts/build_report.py --format docx`
  - 两者都要 → `python scripts/build_report.py --format all`
- **不主动生成网页版**（本技能无 HTML 输出）。

## ⑦ 输入输出

**输入**：器具清单（名称/编号/上次校准日/周期）或动作指令。

**输出（默认）**：MD 管理台账 `metrology_report.md`，含字段：
器具名称、编号、类别、上次校准、周期(月)、下次校准、状态、送检建议。

**输出（按需导出）**：经用户确认后，可生成 **Word(.docx)** / **Excel(.xlsx)** 版本（状态列带颜色标注），文件名 `metrology_report.docx` / `metrology_report.xlsx`。

**命令示例**
```bash
python scripts/build_report.py                        # 仅 MD（默认）
python scripts/build_report.py --format xlsx         # 导出 Excel
python scripts/build_report.py --format docx          # 导出 Word
python scripts/build_report.py --format all           # MD + DOCX + XLSX
python scripts/build_report.py --data-file instruments.json   # 自定义台账
```

**示例（MD 节选）**
```markdown
| 器具名称 | 编号 | 上次校准 | 周期(月) | 下次校准 | 状态 |
|----------|------|----------|----------|----------|------|
| 数显卡尺 0-150mm | CL-001 | 2026-01-15 | 12 | 2027-01-15 | 正常 |
| 千分尺 0-25mm | CL-002 | 2025-06-20 | 12 | 2026-06-20 | 超期(红色) |
```

## ⑧ 内置知识 / 参考清单

- **校准周期默认参考（非强制，以企业制度为准）**：
  - 长度类（卡尺/千分尺/百分表）：12 个月
  - 温度类（热电偶/热电阻）：12 个月
  - 压力类（压力表/压力变送器）：6 个月（强检压力表按法规）
  - 电气类（万用表/兆欧表）：12 个月
  - 力学类（砝码/天平）：12 个月
- **三级预警规则**：超期=立即停用；≤7 天=本周送检；≤30 天=尽快送检；>30 天=正常。
- **适用标准类别（推荐，需人工核对现行有效版本）**：JJG 30（卡尺）、JJG 21（千分尺）、JJG 229（铂热电阻）等。
- **MSA 关联**：关键测量设备建议按控制计划要求执行 GRR 分析。

> 注：周期为通用建议，企业具体周期、强检目录、证书编号以实际制度与证书为准，未提供处标注「待企业补充」。

## ⑨ 联动声明（纯提示）

- 与 `qms-form-builder`（表单生成）：SIP/SOP 中量具需标注校准有效期，可联动核查超期。
- 与 `nonconformance-handling`（不合格处理）：超期量具测得的数据可能涉及不合格品追溯。
- 与 `nc-audit`（内审不符合项）：量具超期是内审常见严重不符合项（条款 7.1.5）。
- 与体系文件类技能：计量台账与 MSA 计划可纳入体系文件。
